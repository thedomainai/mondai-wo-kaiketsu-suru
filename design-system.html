Reference

Governance Rules

トークン・コンポーネントの運用ルール。一貫性を保つための制約。

  1. 01content slide は data-attribute + shared runtime で header/footer を出す。
  2. 02新規作成 / 大幅改修は ds-stage + ds-grid で section-first layout を組む。
  3. 03agenda は manifest source から row を導出する。
  4. 04divider と center-title は既存 class/data contract を維持する。
  5. 05summary / closing / cta / qa は custom kind だが、background / footer meta / visual family は shared token に揃える。
  6. 06ハードコード値の禁止: color: rgba(0,0,0,.34) ではなく color: var(--ink-34) を使う。
  7. 07スペーシングは親の gap に固定する。子の margin は使わない。不均一な距離が必要な場合はサブコンテナを挟む。
  8. 08optical gap 指示は受け入れない。スペーシングスケール上の値に丸める。中間値の需要が 3 枚以上で出現した場合はスケールを見直す。
  9. 09新規コンポーネントは 3 枚以上で同じパターンが出現したら ds-* クラスに昇格させる。
  10. 10装飾 border-top / border-left の禁止。枠の一辺だけを太線・アクセント色で強調するパターンは使わない。強調したいときは、塗り・タイポ・バッジ・全周の細い枠で処理する。
