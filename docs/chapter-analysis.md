# 問題を解決する — 章構成分析

## オープニング（slide_01〜08）

- 目的
  - 講義本編に入る前に「聞く準備」を整える。登壇者への信頼を確立し、研修が存在する理由を示し、全体の地図を渡すことで、受講者が「どこに向かって聞けばいいか」を理解した状態で本編に入れるようにする。
- 構成
  - カバー（slide_01）
    - 目的
      - 研修の表紙として、タイトル・サブタイトル・実施情報を提示する。
    - 章目的との接続
      - これから始まる研修のテーマと場の公式性を最初に固定する。
    - 要素
      - Core Message
        - これから「問題を解決する」をテーマにした研修が始まる。
      - Support
        - Evidence
          - タイトル、サブタイトル、実施日、担当者を明示し、研修の文脈を定める。
      - Audience Action
        - 研修の主題と場の前提を受け取る。
      - Transition
        - 次に「誰から学ぶか」を示し、聞く準備を進める。
    - 表現
      - Text
        - タイトル、サブタイトル、実施日、担当者
      - Visual
        - 表紙レイアウト
      - Interaction
        - 特になし
  - プロフィール①（slide_02）
    - 目的
      - 登壇者のファクト（役割・経歴）を示し、信頼の根拠を提供する。
    - 章目的との接続
      - 研修内容を語る資格と経験の土台を示す。
    - 要素
      - Core Message
        - 登壇者は問題解決を実務で扱ってきた人物である。
      - Support
        - Evidence
          - 氏名・役職、当社での役割、経歴の年齢バーを示す。
        - Example
          - 経営管理を起点とした「戦略策定 / 戦略の係数化と管理 / 達成支援」の役割ツリー。
      - Audience Action
        - 話し手の実務背景を理解する。
      - Transition
        - 次に、ファクトだけでなく価値観も示す。
    - 表現
      - Text
        - 氏名・役職、役割説明、経歴ラベル
      - Visual
        - 年齢バー、役割ツリー
      - Interaction

      - 登壇者の価値観を示し、研修の背景にある思想への共感を促す。
    - 章目的との接続
      - 本編で扱う原理原則のトーンを先に共有する。
    - 要素
      - Core Message

        - Example

## 第1章 ワーク（slide_09〜14）

- 目的
  - 研修前の「自分のロジックツリーの精度」を受講者に自覚させる。自分が能力不足であることを自覚することで、以降の内容が「自分ごと」として入ってくるようにする。
- 構成
  - 章アジェンダ（slide_09）
    - 目的
      - 受講者に現在地（第1章）を示す。
    - 章目的との接続
      - 体験パートが研修の出発点であることを示す。
    - 要素
      - Core Message
        - いまから第1章のワークに入る。
      - Support
        - Evidence
          - 全10章のうち第1章をアクティブ表示する。
      - Audience Action
        - これから体験パートに入る心構えを持つ。
      - Transition
        - 章タイトルを経て、すぐに実際のワークへ進む。
    - 表現
      - Text
        - 章名、全体アジェンダ
      - Visual
        - アクティブ表示付きリスト
      - Interaction
        - 特になし
  - 章タイトル（slide_10）
    - 目的
      - 章の区切りを視覚的に示す。
    - 章目的との接続
      - ここからは知識の説明ではなく、まず自分で考える章であることを切り替える。
    - 要素
      - Core Message
        - 第1章はワークである。
      - Support
        - Evidence
          - 章タイトルを単独で提示する。
      - Audience Action
        - 聞くモードから考えるモードへ切り替える。
      - Transition
        - 次のスライドで実際の問いと進め方を示す。
    - 表現
      - Text
        - 「第01章 ワーク」
      - Visual
        - タイトルスライド
      - Interaction
        - 特になし
  - ケーススタディワーク（slide_11）
    - 目的
      - 受講者に「美味しいハンバーガーをつくるには？」という問いを自力で分解させ、現在の思考の癖を可視化する。
    - 章目的との接続
      - 章目的である「自分の思考の癖の自覚」を実際に発生させる中心スライド。
    - 要素
      - Core Message
        - まずは自分のやり方で問いを分解してみる。
      - Support
        - Evidence
          - ワークの手順を3ステップで明示する。
        - Example
          - 「美味しいハンバーガーをつくるにはどうすべきか？」という共通ケースを置く。
      - Audience Action
        - 個人で考え、メモに書き出し、自分の思考の癖を観察する。
      - Transition
        - ワーク後に、どう考えたかを振り返る。
    - 表現
      - Text
        - お題、補足指示、プロセス文
      - Visual
        - Taskカード、Processカード、補足帯
      - Interaction
        - 個人ワーク
  - 振り返りの問いかけ（slide_12）
    - 目的
      - ワーク直後に沈黙を作り、受講者が自分の思考を振り返る間を生む。
    - 章目的との接続
      - 体験を単なる作業で終わらせず、自覚に変える。
    - 要素
      - Core Message
        - 自分が何を見て、どう考えたかを振り返ることが重要である。
      - Support
        - Evidence
          - 「見えている対象と、そこから打てる手を比べる」という観点を補足する。
      - Audience Action
        - 自分の考え方を内省する。
      - Transition
        - 次に、考え方の差をA/B比較で見せる。
    - 表現
      - Text
        - 問い、補足文
      - Visual
        - WORKラベル、余白を生むレイアウト
      - Interaction
        - 内省
  - 思考の差の構造（slide_13）
    - 目的
      - AさんとBさんの対比で「物から分解する」と「体験から分解する」の差を見せ、分解の起点が解の品質を決めることを示す。
    - 章目的との接続
      - ワークで生まれた差を言語化し、自分の思考の癖を外から見える形にする。
    - 要素
      - Core Message
        - 何を見て分解するかの違いが、出てくる解の質を左右する。
      - Support
        - Comparison
          - Aさんは「物」から、Bさんは「体験」から分解していると対比する。
        - Evidence
          - 思考の差は「範囲」と「粒度」の差異から生まれると整理する。
      - Audience Action
        - 自分がA/Bどちらに近いかを照らし合わせる。
      - Transition
        - 次に、この差をより抽象化して一般原則に上げる。
    - 表現
      - Text
        - 導入文、結論帯
      - Visual
        - A/Bカード、強調枠
      - Interaction
        - 自己照合
  - 思考の差の抽象化（slide_14）
    - 目的
      - A/B/Cの3段階比較でメンタルモデルの違いを抽象化し、「見える要素の範囲と構造」が解の品質を決めるという原則を確立する。
    - 章目的との接続
      - 個人の癖の話を、後続章でも使える一般原則に変換する。
    - 要素
      - Core Message
        - 見えている要素の範囲と構造が広く正確なほど、解の品質は高くなる。
      - Support
        - Comparison
          - RANDOM / PARTIAL TREE / FULL TREE の3段階比較。
        - Evidence
          - 評価軸として「範囲/粒度」と「つながり」を明示する。
      - Audience Action
        - 自分の思考がどのレベルにあるかを認識する。
      - Transition
        - 次章で「問題解決能力とは何か」を定義する。
    - 表現
      - Text
        - 導入文、評価軸帯
      - Visual
        - 3段階のSVG図、強調枠
      - Interaction
        - 自己評価

## 第2章 前提・背景（slide_15〜21）

- 目的
  - 「問題解決能力が高い」とはどういう意味かを定義し、研修のテーマである「再現性」という概念を確立する。第3章以降の本論に共通する前提認識を全員で揃えることで誤解の可能性を下げる。
- 構成
  - 章アジェンダ（slide_15）
    - 目的
      - 受講者に現在地（第2章）を示す。
    - 章目的との接続
      - ここからは原理の前提を揃える章だと示す。
    - 要素
      - Core Message
        - いまから前提・背景を共有する。
      - Support
        - Evidence
          - 全10章のうち第2章をアクティブ表示する。
      - Audience Action
        - 体験から定義のフェーズへ切り替える。
      - Transition
        - 章タイトルを経て、まず問題解決能力の定義に入る。
    - 表現
      - Text
        - 章名、全体アジェンダ
      - Visual
        - アクティブ表示付きリスト
      - Interaction
        - 特になし
  - 章タイトル（slide_16）
    - 目的
      - 章の区切りを視覚的に示す。
    - 章目的との接続
      - 本論に入る前に、共通前提を整える章であることを固定する。
    - 要素
      - Core Message
        - 第2章は前提・背景である。
      - Support
        - Evidence
          - 章タイトルを単独で提示する。
      - Audience Action
        - ここから定義と前提の共有に入る認識を持つ。
      - Transition
        - 次のスライドで、問題解決能力の定義を示す。
    - 表現
      - Text
        - 「第02章 前提・背景」
      - Visual
        - タイトルスライド
      - Interaction
        - 特になし
  - 問題解決能力の定義（slide_17）
    - 目的
      - 「問題解決能力が高い」という言葉の定義を最初に確立し、この研修が扱う能力の輪郭を示す。
    - 章目的との接続
      - 研修全体のコア概念を最初に定義し、以後の章の評価軸を揃える。
    - 要素
      - Core Message
        - 問題解決能力とは、正しい問いを立て、分解し、仮説を置き、検証し、解を伝える力である。
      - Support
        - Evidence
          - 補足として、問いの質・分解の質・仮説の質・検証の速さと正しさ・伝達の質が再現性を決めると示す。
      - Audience Action
        - 以後の内容を5工程のレンズで捉える。
      - Transition
        - 次に「能力が高い」とは期待値の話だと補足する。
    - 表現
      - Text
        - QUESTIONラベル、問い、答え、補足帯
      - Visual
        - 問いと答えを対置するレイアウト
      - Interaction
        - 特になし
  - 期待値という本質（slide_18）
    - 目的
      - 「必ず解ける＝高い能力」という誤解を解き、「期待値を高めること」が本質だと示す。
    - 章目的との接続
      - 問題解決能力を「必勝」ではなく「再現性」に接続する。
    - 要素
      - Core Message
        - 能力の高さとは、どんな問題でも解けることではなく、解ける可能性を高め続けることである。
      - Support
        - Comparison
          - Wrong「必ず解ける」と Right「解ける可能性が高い」の対比。
        - Evidence
          - 1回ではなく100回の平均で設計するという補足。
      - Audience Action
        - 問題解決を確率の観点で捉える。
      - Transition
        - 次に、結果だけで行動を評価しない前提を置く。
    - 表現
      - Text
        - Wrong/Right、本質ブロック、補足帯
      - Visual
        - 色分けカード
      - Interaction
        - 特になし
  - 結果と行動の切り離し（slide_19）
    - 目的
      - 「結果が悪い＝行動が悪い」という評価軸の誤りを示し、コントロールできるプロセスを学習対象にすべきだと示す。
    - 章目的との接続
      - 再現性を上げるには、評価対象を結果ではなく意思決定と行動に置く必要があると示す。
    - 要素
      - Core Message
        - 学習対象にすべきなのは、結果そのものではなく、意思決定と行動の品質である。
      - Support
        - Comparison
          - 「結果が悪い→行動が悪い」という誤りと、「評価すべきは意思決定の品質・実行の品質」という正しい見方の対比。
        - Evidence
          - 結果は市場環境や他者判断など多要因で決まるという結果マップ図。
      - Audience Action
        - 結果への短絡的な反応を一度止める。
      - Transition
        - 次に、再現性をどう捉えるかを3原則で定義する。
    - 表現
      - Text
        - リード文、Wrong/Right説明、結語
      - Visual
        - 結果マップ図
      - Interaction
        - 特になし
  - 再現性の3原則（slide_20）
    - 目的
      - 「再現性とは正しく間違えること」という定義を3原則に分解し、どう失敗と向き合うかの指針を示す。
    - 章目的との接続
      - 第2章の中心概念である再現性を、実際の行動規律に落とす。
    - 要素
      - Core Message
        - 再現性とは、失敗しないことではなく、正しく間違え、早く戻り、致命傷を避けることである。
      - Support
        - Evidence
          - 失敗の定義と3原則の説明・図解を置く。
        - Example
          - 誤り認識、適切な場所への復帰、致命的失敗回避の各パターン。
      - Audience Action
        - 失敗観を「避けるもの」から「扱うもの」へ切り替える。
      - Transition
        - 最後に、この再現性と論理的思考を接続する。
    - 表現
      - Text
        - 定義ストリップ、原則説明、結語
      - Visual
        - ステップ図、分岐図、ハザード図
      - Interaction
        - 特になし
  - 研修テーマへの接続（slide_21）
    - 目的
      - 「再現性高く問題を解決する」という課題と「論理的思考」という方法を接続し、この研修が何を学ぶ場かを宣言する。
    - 章目的との接続
      - 前提認識を、これから学ぶ技術の全体像へ橋渡しする。
    - 要素
      - Core Message
        - 論理的思考は、再現性高く問題を解決するための方法の1つである。
      - Support
        - Evidence
          - 問いを定義し、論点を分解し、仮説を作り、検証するという方法を要約する。
      - Audience Action
        - 以後の研修内容を「問題解決の技術」として捉える。
      - Transition
        - 次章で、その問題解決プロセスの全体地図を渡す。
    - 表現
      - Text
        - START/METHODブロック、補足帯
      - Visual
        - 左右ブロックと矢印バッジ
      - Interaction
        - 特になし

## 第3章 問題解決の概要（slide_22〜28）

- 目的
  - 問題解決の「全体地図」を渡すことで、第4〜9章の各詳細が「全体の中のどこか」として理解できるようにする。
- 構成
  - 章アジェンダ（slide_22）
