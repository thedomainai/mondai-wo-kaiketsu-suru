# Header Copy Revision Ver2

## Premise

- `header--title` (`data-header-title`) = main message
- `header--subtitle` (`data-header-subtitle`) = slide title
- This document is a proposal set. It does not rewrite slide HTML by itself.

## Ver2 Rules

### Semantic rules

1. Main message is the slide's answer, judgment, or takeaway.
2. Slide title is the node label that locates the slide inside the deck or logic tree.
3. Main message and slide title must not be reversible.
4. Main message should answer "so what?" by itself.
5. Slide title should answer "what topic is this slide about?" by itself.

### Writing rules

1. Prefer sharp business phrasing over explanatory prose.
2. Prefer short declarative expressions or noun-ending phrases where they stay clear.
3. Avoid generic labels such as `重要な点`, `〜とは何か`, `内容設計` alone unless a stronger node label is impossible.
4. Avoid duplication between main message and slide title.
5. Avoid filler such as `〜について`, `〜することが大切`, `〜が重要`.
6. Use explicit business verbs for action, judgment, priority, and causality.

### Deck-level consistency rules

1. Summary slides: main message = chapter takeaway, slide title = `第XX章まとめ`.
2. Principle slides: slide title carries the taxonomy (`原則1`, `失敗2`, etc.), main message carries the rule.
3. Error-pattern slides: slide title names the error, main message names the diagnostic or corrective logic.
4. Adjacent slides in the same chapter should use parallel grammar where possible.

### Self-check

- If the main message is shown alone, a reader can still infer the slide's point.
- If the slide title is shown alone, a reader can still place the slide in the deck structure.
- The pair has a clear role split: `message = claim`, `title = label`.
