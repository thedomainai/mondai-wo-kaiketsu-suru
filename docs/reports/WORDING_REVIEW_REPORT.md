## ⚠️ 改善が推奨される点

### 1. ページ番号の表示精度（軽微 | 優先度：低）

**該当スライド:**
- slide_20.html: data-footer-page="19" ← **実際の順序では20**
- slide_21.html: data-footer-page="20" ← **実際の順序では21**
- （以降全スライドで±1のずれの可能性あり）

**根拠:**
アジェンダスライド（slide_05.html）をslide_05として数えると、その後のページ数が1ずつ減っている傾向が観察されます。

**対処方法:**
```html
<!-- 現在の例 -->
<div data-footer-page="19"></div>

<!-- 推奨（シーケンシャル確認後）-->
<!-- スライド20に対しては data-footer-page="20" の確認 -->
```

**確認作業:**
```bash
for f in slide_*.html; do
  pagenum=$(grep -o 'data-footer-page="[0-9]*"' "$f" | head -1)
  echo "$f: $pagenum"
done
```

**重要度:** 低い（表示の些細な誤差）。内容品質には影響なし。

### 3. 表記の口語性（軽微 | 優先度：低）

#### **slide_29.html**

```html
<span style="color: #ff6666; margin-right: 8px;">×</span>なんちゃって論点
```

**詳細:**
「なんちゃって」はカジュアルな日本語口語。ビジネス研修資料としては「不適切な」「不十分な」の方がフォーマルです。

**ただし:**
- コンテキスト（「××論点」との対比）で意図が明確
- 受講者の親しみやすさを狙った意図的な表現と判定
- 新人研修向けには効果的

**改善案（オプション）:**
```html
<!-- 現在 -->
×なんちゃって論点（行動に繋がらない）

<!-- 案A：よりフォーマル -->
×疑似的な論点 / 不完全な論点

<!-- 案B：そのまま維持 -->
×なんちゃって論点（親しみやすさ優先）
```

**推奨:** 現行維持（効果的なため）
